## Тема ВКР

**Анализ и генерация аргументации на основе больших языковых моделей**

Репозиторий содержит программные материалы к выпускной квалификационной работе на тему **«Анализ и генерация аргументации на основе больших языковых моделей»**.

Работа посвящена исследованию применения больших языковых моделей для двух связанных задач обработки естественного языка:

1. классификации позиции текстового высказывания относительно заданного тезиса;
2. генерации коротких аргументов с заданной позицией;
3. автоматизированной оценки генерации с использованием подхода LLM-as-a-Judge.

## Цель проекта

Цель проекта — реализовать и экспериментально проверить prompt-based подход к анализу и генерации аргументации на русском языке с использованием открытовесовых и закрытых API-моделей.

## Основные задачи

В рамках проекта реализованы:

* подготовка русскоязычного корпуса RuArg к формату `текст – тема – тезис – метка`;
* формирование сбалансированной оценочной выборки для классификации;
* запуск классификационных экспериментов для локальных и API-моделей;
* проведение абляционного анализа prompt-шаблонов;
* генерация аргументов по заданным темам и позициям;
* автоматизированная оценка генерации с помощью двух judge-моделей;
* агрегация результатов, построение таблиц и графиков;
* диагностика форматных артефактов в ответах моделей.

## Используемые данные

В работе использовался русскоязычный корпус **RuArg**, содержащий пользовательские тексты по темам:

* ношение масок;
* введение и соблюдение карантина;
* вакцинация.

Для классификации использовались три класса:

* `AGAINST` — аргумент против заданного тезиса;
* `NO_ARG` — отсутствие явного аргумента относительно тезиса;
* `FOR` — аргумент в поддержку заданного тезиса.

## Описание ноутбуков

### `Local.ipynb`

Ноутбук предназначен для запуска локальных экспериментов через LM Studio. В нем реализованы:

* подключение к локальному OpenAI-совместимому API;
* классификация текстов;
* генерация аргументов;
* сохранение ответов моделей;
* расчет метрик;
* построение графиков.

### `Cloud.ipynb`

Ноутбук предназначен для запуска экспериментов через OpenRouter API. В нем реализованы:

* подключение к API;
* запуск закрытых API-моделей;
* классификационные эксперименты;
* генеративные эксперименты;
* сохранение результатов для последующего объединения.

### `Combined_Judge.ipynb`

Ноутбук предназначен для объединения локальных и API-результатов. В нем реализованы:

* объединение сгенерированных аргументов;
* диагностика форматных артефактов;
* LLM-as-a-Judge оценка;
* расчет ensemble-оценки;
* формирование итоговых таблиц и графиков.

## Метрики

Для классификации использовались:

* `accuracy`;
* `F1` по каждому классу;
* `macro-F1`.

Для генерации использовались критерии:

* релевантность тезису;
* соответствие требуемой позиции;
* аргументативность;
* связность;
* фактическая осторожность;
* чистота формата.

Также отдельно анализировались форматные артефакты:

* пустые ответы;
* слишком короткие ответы;
* англоязычные служебные маркеры;
* markdown-разметка;
* повтор prompt;
* reasoning-блоки.
